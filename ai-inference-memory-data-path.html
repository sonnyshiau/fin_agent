LLM Inference · Memory Hierarchy

一個 Token，如何穿過 GPU 的 Memory Data Path

切換 Prefill、Decode、KV Hit 與 KV Miss，觀察 Weights、KV Cache 與 Activation 在 On-chip SRAM、GPU HBM、Host DRAM、NVMe 之間如何流動。

Prefill：算得多大量 prompt tokens 形成大 GEMM，權重重用率與 arithmetic intensity 較高。
Decode：搬得多每輪只產生少量 token，反覆讀 weights 與歷史 KV，容易受 HBM bandwidth 限制。
Offload：容量換延遲DRAM／NVMe 擴大有效容量，但 KV 必須 reload 回 HBM 才能高效 decode。

Memory Temperature

越靠近 Compute，越快、越貴、容量越小

On-chip SRAM當下 tile 與 working set；最低 latency、最小容量。
GPU HBMWeights 與 active KV；即時 decode 的主要 bandwidth 層。
Host DRAMWarm KV 與 staging；容量較大，但 reload 有額外 hop。
NVMe SSDCold KV；大容量，必須預取才能避免 decode stall。
